「性格」不再是靜態的形容詞標籤（如「仁義」或「狡詐」），而是被精確定義為一組動態的「響應曲線（Response Curves）」與「決策函數」
一、 [[性格]]即「對價斜率（Trade-off Slopes）」

不同性格的模板，其最核心的差異在於**參數轉換的「匯率」或「斜率」不同**。當面臨同樣的環境壓力（TNS）時，性格曲線決定了角色願意犧牲什麼來換取什麼


二、 非線性相變與「臨界點（Thresholds）」

人類的性格往往不是一條平滑的直線，而是存在極限的**「階躍函數（Step Function）」或「S型曲線」**。
- **平庸狀態（低值區間）**：當好感度或羈絆（BND）處於一般水準時，角色的響應曲線受社會規範約束，行為表現為「客套」或「理性」，此時採取極端行動的機率趨近於零。
- **一怒為紅顏（臨界跳變）**：當羈絆（BND）突破特定臨界點（如大於 0.9），或是張力（TNS）破表時，性格曲線會發生**「斜率驟變」**。角色會瞬間無視資源與健康的損耗，觸發「報復」或「瘋狂」的極端對價運動（例如劉備為關羽不計代價起兵復仇）。

三、 L1 狀態與 L2 脾氣的解耦

在系統架構中，這兩者被嚴格分開：

- **L1 參數**：代表角色的當前狀態（如現在有多少資源、多大壓力），像是角色的「生理指標」。
- **L2 性格響應曲線**：代表角色面對 L1 數值變化時的「條件反射算式（If-Then 邏輯）」，像是角色的「脾氣」或「決策演算公式」。

角色的「目標」不再是感性的文學標籤（如「匡復漢室」或「一統天下」），而是被徹底轉化為**「參數位能差的補完運動」**與精確的**「對價支付策略」**。

**一、 目標即「位能差」與驅動力公式** 角色的運動本質上是為了抹平當前參數與目標參數之間的「壓力差」。引擎將這個驅動力定義為一個數學公式：D=∑wi​⋅(Ptarget,i​−Pcurrent,i​)。 其中 Ptarget​ 是角色渴望達到的參數目標，Pcurrent​ 是當前參數，而 wi​ 是由性格常數決定的權重。如果算出的驅動力 D 很大，角色就會產生劇烈的運動；如果 D 趨近於零，角色就會進入舒適圈並停止運動。

**二、 JSON 中的** **Drive_Vector** **實作** 在系統的設定與 JSON 模板中，目標被編譯為明確的 `Drive_Vector`（動機向量）。

- 以玄奘的「西天取經」為例，系統不會使用文字描述，而是將其參數化為：`{"Target": "INF", "Direction": "Minimize", "Intensity": 0.98}`。這代表他的運動向量恆定指向減少情報熵（即揭露未知與真相）。
- 曹操的「稱霸天下」則會被定義為追求個人力量位能（POW）的最大化，以及將發散的資源（RES）強制收斂到自己的座標點。

**三、 目標作為一種「支付策略」** 目標不僅定義了角色想往哪裡移動，更綁定了角色的**「對價支付順序 (Priority of Sacrifice)」**。 角色為了達成目標向量，會主動選擇願意支付什麼樣的代價。例如，劉備的目標是維護漢室的集團資源與秩序，為此他極度願意支付個人的自尊（PRD）與安全感（SAF）。當目標方向與支付能力發生嚴重衝突，或是發現支付了所有資源仍無法達成目標時，角色的張力（TNS）會超載，進而觸發靈魂的「相變」（如劉備從守護者跳變為瘋狂的復仇者）。

**四、 升級為對抗性戰略向量（零和博弈）** 在引擎的進階邏輯中，為了落實能量守恆，目標被進一步升級為**「對抗性的決策算式」**，使其具備幾何上的矢量方向。

- **掠奪型向量**：曹操的目標不僅是增加自身的數值，其算式可能被定義為 `魏.POW += (漢.RES + 蜀.RES) * 效率`，這意味著他的擴張必須從敵對法人身上「抽血」。
- **清理型向量**：對於像關羽這樣的「協助者」，他的目標不是增加自己的權力，而是直接指向敵對武將的安全感（SAF）。他透過消耗自己的資源來削減敵方防禦，藉此降低主公（劉備）前進時的環境阻力
