「補償機制」與「客製化故事」是將這套系統從單純的「物理模擬器」昇華為「開放式敘事沙盒」的核心功能。這兩者結合，讓使用者能夠以上帝視角（造物主）的身份，精確地引導並重塑故事的走向。

以下是這兩項機制的運作說明與來源：

一、 補償機制（使用者干預 / 神之手）

補償機制（Deus Ex Machina Layer）允許使用者在系統自動演化的過程中，強行注入外部變數（如：幸運道具、奇遇、天災），以此來改變角色的運動軌跡與世界的位能場。

2. **作為「算符（Operator）」改變對價斜率**： 補償機制不是空洞的名詞，而是一組能改變「摩擦力」與「成長率」的數學外掛。
    - 例如給予主角一套「雙修功法」，這在引擎中是一個倍率算符：POWnew​=POWold​+(ΔTraining×1.5)。只要這個 Buff 存在，系統在計算時就必須強制乘以 1.5 倍的收益。
    - 投入「大旱三年」等環境變數，會導致全域資源（RES）損耗率飆升，迫使原本安逸的角色為了生存而支付尊嚴（PRD）。
3. **代價與「世界的惡意」（因果守恆）**： 補償機制並非毫無代價的「強買強賣」。當使用者強行賦予角色「奇蹟」（如非對價型強化）時，系統為了維持能量守恆，會產生「系統性負債」。角色的「世界隸屬度（LOA）」會被強制鎖定，並引發環境阻力係數（μ）升高或周圍敵對角色張力（TNS）暴增（即天劫或全世皆敵的現象）。此外，引擎也會自動生成 [氣運] 或 [因緣] 標籤，以在邏輯上修補這段不合理的干預。

二、 客製化故事（敘事逆向編譯）

如果你想指定一個特定的故事走向，NISA 引擎採用的是**「敘事逆向編譯（Reverse Compilation）」**技術。你不需要親自寫劇本，而是交出一份「意圖藍圖（Intent Blueprint）」，讓 AI 幫你把感性描述轉譯為底層的 JSON 物理數據。

1. **設定終點，倒推軌跡**： 如果你要求一個「主角先苦後甘、結局大家戰死」的故事，AI 會將結局解讀為參數的「收斂點」（例如：全員生命 HLT→0，全域張力 TNS→∞）。接著，系統會自動倒推，為了達到這個悲劇結果，中間必須發生哪些「情報熵（INF）崩潰」或高昂的「對價支付」。
2. **角色初始化的物理錨定**： 系統會根據你的要求自動配給角色的初始參數。例如要求一個堅毅但卑微的少年，AI 會賦予他極低的權力（POW:0.1）與資源（RES:0.05），但給予極高的驅動力（α:0.95）與自律（ϵ:1.0）。
3. **動態調整與重新運算**： 在客製化故事的演化途中，你可以隨時以「觀察員」的身份介入。如果你對演化到一半的劇情不滿意（例如不想讓主角毀滅世界），你可以透過**補償機制**手動修改 JSON 數據裡的變數，4060 本地渲染模型就會立刻根據新的參數走向，轉向渲染出全新的結局。

**總結來說：**「客製化故事」是設定這場物理實驗的起點與終點，而「補償機制」就是你在實驗過程中，用來隨時撥動角色命運、改變環境重力場的「神之手」。
